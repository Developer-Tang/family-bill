# 贡献指南

感谢您对家庭记账系统项目的关注和支持！我们非常欢迎社区成员的贡献，无论是功能开发、Bug修复、文档完善还是提出建议，都能帮助项目不断进步。

## 贡献方式

您可以通过以下几种方式为项目做出贡献：

1. **报告问题**：在GitHub Issues中提交Bug报告或功能建议
2. **提交代码**：通过Pull Request提交代码修改
3. **改进文档**：修正错误、补充内容、优化描述
4. **参与讨论**：回答问题、提供建议、参与功能规划

## 开发规范

### 代码风格

- 后端Go代码：遵循Go语言官方风格指南，使用`gofmt`工具格式化代码
- 前端Vue代码：遵循Vue官方风格指南，使用ESLint进行代码检查
- 提交前请确保代码通过所有测试

### 提交信息规范

提交信息应清晰明了，包含以下内容：

```
<类型>: <简短描述>

<详细说明>（可选）

<关联issue>（可选）
```

其中类型包括：
- `feat`：新功能
- `fix`：Bug修复
- `docs`：文档更新
- `style`：代码风格改进
- `refactor`：代码重构
- `test`：测试相关
- `chore`：构建过程或辅助工具变动

## 提交Bug报告

提交Bug报告时，请包含以下信息：

1. **问题描述**：清晰描述遇到的问题
2. **复现步骤**：详细的复现步骤，便于我们定位问题
3. **期望结果**：描述您期望的正确行为
4. **实际结果**：描述实际发生的行为
5. **环境信息**：系统版本、浏览器类型、项目版本等
6. **截图**：如有可能，请提供相关截图

## 贡献代码流程

1. **Fork项目**：在GitHub上Fork项目到您的账户
2. **克隆代码**：将Fork后的仓库克隆到本地
3. **创建分支**：基于`main`分支创建新的功能分支
4. **开发修改**：在新分支上进行开发和修改
5. **测试验证**：确保所有测试通过
6. **提交PR**：提交Pull Request到原始仓库的`main`分支

### PR提交规范

提交PR时，请提供以下信息：

1. **标题**：简洁明了地描述PR的目的
2. **描述**：详细说明所做的修改、解决的问题等
3. **关联issue**：如有相关issue，请在描述中引用
4. **测试情况**：说明您进行了哪些测试验证

## 文档贡献

文档是项目的重要组成部分，我们欢迎对文档进行以下改进：

1. **修正错误**：修正文档中的错误信息
2. **补充内容**：补充缺失的文档内容
3. **优化描述**：使文档更加清晰易懂
4. **添加示例**：添加代码示例、截图等辅助说明

## 行为准则

参与项目贡献时，请遵守以下行为准则：

1. **尊重他人**：尊重每位贡献者，不进行人身攻击或歧视性言论
2. **建设性反馈**：提供建设性的反馈意见，避免无意义的批评
3. **耐心交流**：耐心解答问题，帮助新人融入社区
4. **关注质量**：注重代码和文档质量，不提交未经测试的代码
